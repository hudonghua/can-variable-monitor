## 先看这个结论

资料里定义的是 CAN 业务：QT 电脑端需要按 `CAN ID + 8字节DATA` 理解内容。CAN_TO_NET 是中间透传工具，不负责解释业务字段。

如果 QT 走 CAN-NET 透传接口，资料里写的是 `192.168.0.105:500`，按 CAN-NET 帧封装 CAN ID 和 DATA。

如果 QT 走你现场调通的 Modbus 接口 `192.168.0.105:503`，本表已经按 CAN ID 补入 CAN_TO_NET 地址段。一个 CAN ID 占 4 个连续地址，对应 8 字节 CAN DATA。

## 地址与CAN字节对应规则

一个 `CAN_TO_NET地址` 可以放 2 个 CAN 字节，低字节在前，高字节在后。

```text
地址值 = 低字节 + 高字节 * 256
```

也就是：

| CAN_TO_NET地址 | 对应CAN字节 | 16位值 |
|---|---|---|
| 起始地址 | B0、B1 | B0 + B1 * 256 |
| 起始地址 + 1 | B2、B3 | B2 + B3 * 256 |
| 起始地址 + 2 | B4、B5 | B4 + B5 * 256 |
| 起始地址 + 3 | B6、B7 | B6 + B7 * 256 |

例如 `CAN ID 0x50` 对应 `CAN_TO_NET地址 100-103`：

| CAN_TO_NET地址 | 对应CAN字节 | 16位值 |
|---:|---|---|
| 100 | B0、B1 | B0 + B1 * 256 |
| 101 | B2、B3 | B2 + B3 * 256 |
| 102 | B4、B5 | B4 + B5 * 256 |
| 103 | B6、B7 | B6 + B7 * 256 |

QT 写入时，把 8 个 CAN 字节按上面规则合成 4 个 `uint16` 写入。QT 读取时，把每个 `uint16` 再拆成低字节和高字节。

## QT同事怎么用

1. 先按资料里的 `CAN ID` 查本表。
2. 看 `CAN信号内容`、`Byte`、`Bit`，确定要读写的业务字段。
3. 如果使用 CAN-NET 透传，直接按 CAN ID 和 8 字节 DATA 封装发送/接收。
4. 如果使用本次封装的 Modbus 503 接口，QT 可以直接调用 `readCanFrame()` / `writeCanFrame()`；封装内部会按本表把 CAN ID 转成 `CAN_TO_NET地址`。

## CAN-NET透传格式摘要

资料说明：CAN-NET 只是透传 CAN 帧，业务仍按 CAN ID 和 8 字节 DATA 解释。帧格式摘要如下：

| 字段 | 含义 |
|---|---|
| 0-1 | 帧头 0xAA 0x55 |
| 2 | 协议版本 0x01 |
| 3 | 帧类型：0x01发送CAN，0x02接收CAN，0x03心跳 |
| 4 | 序号 0x00-0xFF |
| 5 | CAN通道，暂定 0=CAN1 |
| 6 | 帧属性，默认 0x00 标准数据帧 |
| 7 | DLC |
| 8-11 | CAN ID，uint32 小端 |
| 12-19 | CAN DATA，8字节 |
