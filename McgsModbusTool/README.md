# CAN_TO_NET Modbus TCP 电脑端

默认连接参数：

- IP：192.168.0.105
- 端口：503
- 站号：255
- 通讯超时：100ms
- CAN_TO_NET地址偏移：1

## CAN_TO_NET -> 电脑

左侧是读取 CAN_TO_NET 数据区。

常用设置：

```text
功能：03 读保持寄存器
CAN_TO_NET起始地址：61
数量：12
周期：100ms
```

点 `读一次` 可以手动读一帧。点 `自动读` 后按周期读取 CAN_TO_NET 里的数据。

地址规则已经在软件里自动处理。

```text
你只需要填写 CAN_TO_NET地址。
```

默认偏移为 1，也就是：

```text
CAN_TO_NET 1  就填 1
CAN_TO_NET 12 就填 12
```

一般情况下 `地址偏移` 保持默认 `1` 即可。

## 电脑 -> CAN_TO_NET

右侧是发送编辑窗口。每一行就是一个要发给 CAN_TO_NET 的寄存器：

```text
启用 | 名称 | 地址 | 发送值 | 十六进制 | 最近状态
```

例如：

```text
发送1  CAN_TO_NET地址 61  值 12
发送2  CAN_TO_NET地址 62  值 13
发送3  CAN_TO_NET地址 63  值 14
```

点 `发送一次` 会立刻写入 CAN_TO_NET。

点 `自动发送` 会按 `周期(ms)` 持续发送，默认 100ms。

连续地址会自动合并成一包 `16 连续写寄存器`，例如 60、61、62 会合成一包发送。单个地址会用 `06 写单寄存器`。

## 速度建议

如果要求 100ms 内响应：

- 通讯超时保持 100ms 或更低。
- 自动发送周期设置 100ms。
- 如果读和写都开 100ms，实际会共用同一条 TCP 连接串行发送；若写入优先级更高，建议读周期设置为 200ms 或更慢。
- CAN_TO_NET里必须配置好对应地址，否则会返回 Modbus 异常或超时。
